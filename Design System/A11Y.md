## 2. The 4 Principles (POUR)

Per WCAG, all a11y requirements stem dari 4 prinsip:

### 2.1 Perceivable
Informasi harus **bisa diperceive** oleh semua sense. Tidak boleh hanya warna, hanya suara, hanya visual.

**Implications:**
- Every `<img>` has `alt`
- Every icon button has `aria-label`
- Color not sole indicator (pair with icon/text)
- Contrast ≥ 4.5:1 body, 3:1 large text (WCAG AA)
- Transcripts for audio/video (future)

### 2.2 Operable
Interface harus bisa **dioperasikan** lewat multiple input modalities.

**Implications:**
- 100% keyboard navigable (Tab, Enter, Space, Arrow, Esc)
- No keyboard traps (user can always escape)
- Skip links (bypass repetitive nav)
- Sufficient time (no auto-dismiss critical info)
- Touch targets ≥ 44×44px (WCAG AAA, we target this)

### 2.3 Understandable
Content dan UI harus **bisa dipahami**.

**Implications:**
- Consistent nav across pages (InsightSphere: Sidebar + Header always same)
- Clear labels + error messages (Bahasa Indonesia clear, no jargon)
- Predictable behavior (no unexpected context changes on focus)
- Help available (tooltip, inline help text)

### 2.4 Robust
Kompatibel dengan current + future assistive technologies.

**Implications:**
- Valid HTML5
- Semantic elements (`<nav>`, `<main>`, `<button>`, `<section>`)
- Progressive enhancement
- ARIA attributes correct (don't misuse)

## 3. Keyboard Navigation

### 3.1 Standard Key Bindings
| Key | Action |
|---|---|
| `Tab` | Move focus forward to next interactive element |
| `Shift+Tab` | Move focus backward |
| `Enter` | Activate button, submit form, open link |
| `Space` | Toggle checkbox/switch, activate button (not link), page down |
| `Esc` | Close modal, dropdown, popover, tooltip |
| `Arrow ↑ ↓` | Navigate items (menu, listbox, tabs, radio group) |
| `Arrow ← →` | Horizontal nav (tabs, carousel) |
| `Home` / `End` | First / last item in menu/list |
| `PageUp` / `PageDown` | Scroll |
| `/` | Focus search (future enhancement) |
| `Ctrl+K` / `⌘K` | Command Palette |

### 3.2 Wajib Keyboard Support Per Component
| Component | Required Keys |
|---|---|
| Button | Enter, Space |
| Link | Enter |
| Checkbox | Space |
| Radio | Arrow ↑↓, Space |
| Switch | Space, Enter |
| Select (native) | Arrow ↑↓, Enter, Esc |
| Modal | Esc (close), Tab (cycle within) |
| Drawer | Esc (close), Tab (cycle within) |
| Dropdown/Menu | Arrow ↑↓, Enter (select), Esc (close), Home/End |
| Tabs | Arrow ←→, Home/End, Enter/Space |
| Tooltip | Focus shows, Esc hides |
| Combobox | Arrow ↑↓, Enter, Esc, typing filters |
| Tree | Arrow (all 4), Enter, Space |
| Sidebar nav | Tab through, Arrow nav optional |

### 3.3 No Keyboard Trap Rule
User **harus selalu bisa escape** dari any focused widget pakai **keyboard saja**:
- Modal → Esc atau Tab ke close button
- Dropdown → Esc atau Tab out
- Embedded iframe → Tab out (focusout)

**Exception:** Modal focus trap (Radix Dialog) = **intentional** — user harus interact dengan modal sebelum escape. Esc = escape hatch.

### 3.4 Tab Order = DOM Order
Jangan pakai `tabindex="1"`, `tabindex="2"`, dst. (anti-pattern). Tab order **otomatis** dari DOM source order.

**Allowed tabIndex values:**
- `tabIndex={0}` — include in tab order (for `<div role="button">` hack — but prefer real `<button>`)
- `tabIndex={-1}` — focusable via script (`element.focus()`) but skipped in tab order (for custom focus management)

### 3.5 Radix Primitives Handle Keyboard Automatically
Kalau pakai Radix (via shadcn):
- `<Dialog>`, `<DropdownMenu>`, `<Tabs>`, `<Popover>`, `<Tooltip>`, `<Select>`, dll.
- **Keyboard nav built-in** — jangan reinvent dengan `onKeyDown` manual.

Audit finding: Header store selector reinvents keyboard nav (60 LOC) — **migrate ke `<DropdownMenu>`** (lihat `OVERLAYS.md §10.2`).

## 4. Focus Management

### 4.1 Visible Focus Ring — WAJIB
Every interactive element **wajib** punya visible focus indicator. Default browser outline tidak cukup (inconsistent, ugly).

**Standard pattern:**
```tsx
className={cn(
  "outline-none",
  "focus-visible:ring-2 focus-visible:ring-indigo-500 focus-visible:ring-offset-2",
  "dark:focus-visible:ring-offset-slate-900"
)}
```

**Key points:**
- `outline-none` removes default browser outline (we replace, not remove)
- `focus-visible:` — hanya show ring saat keyboard focus (bukan mouse click)
- `ring-2` — 2px ring
- `ring-indigo-500` — brand color
- `ring-offset-2` — gap antara element + ring
- `dark:ring-offset-slate-900` — match dark mode bg

### 4.2 `focus-visible:` vs `focus:` — Policy
| Pattern | Use When |
|---|---|
| `focus-visible:ring-*` | **Default** — show ring only on keyboard |
| `focus:ring-*` | ❌ Legacy — migrate away |
| `focus-within:ring-*` | Container ring when child focused (e.g., input wrapper) |

**Migration:** 52 legacy `focus:ring-*` → `focus-visible:ring-*`.

### 4.3 Focus Ring Color by Context
| Context | Ring Color |
|---|---|
| **Default** | `ring-indigo-500` |
| Destructive action | `ring-rose-500` |
| On dark bg (sidebar) | `ring-white/40` |
| On colored bg (indigo-600) | `ring-white/60` |

### 4.4 Auto-Focus Policy
Use `autoFocus` sparingly:
- ✅ **Modal opens → first input** (Radix Dialog handles)
- ✅ **Search opens → search input**
- ✅ **Error state → error summary** (programmatic)
- ❌ **Page load → arbitrary input** (confusing, scroll jumps)
- ❌ **Multiple autoFocus** dalam 1 view (only 1 wins, unpredictable)

### 4.5 Focus Restoration
When modal/drawer closes, **focus kembali** ke trigger element. Radix handles this automatically. Jangan override.

### 4.6 Programmatic Focus
```tsx
// Correct pattern
const inputRef = useRef<HTMLInputElement>(null);
useEffect(() => {
  if (shouldFocus) inputRef.current?.focus();
}, [shouldFocus]);

<input ref={inputRef} />
```

Jangan pakai `document.getElementById(...).focus()` — tidak React-idiomatic.

### 4.7 Skip Link (Bypass Nav)
**Currently 0 skip links — critical gap.**

Pattern:
```tsx
// Pertama di DOM body / layout
<a
  href="#main-content"
  className={A11Y.skipLink}
>
  Lompat ke konten utama
</a>
```

```tsx
// Di PortalTemplate main area
<main id="main-content" className="flex-1 overflow-y-auto">
  {children}
</main>
```

`A11Y.skipLink` token:
```
sr-only focus:not-sr-only focus:absolute focus:top-4 focus:left-4
focus:z-50 focus:px-4 focus:py-2 focus:rounded-xl
focus:bg-indigo-600 focus:text-white focus:shadow-lg
focus:outline-none focus-visible:ring-2 focus-visible:ring-offset-2 focus-visible:ring-indigo-300
```

Hanya terlihat saat focused via keyboard — tidak mengganggu mouse users.

## 5. Screen Reader Patterns

### 5.1 Screen Reader = What You Can't See
Blind users + low-vision users pakai VoiceOver (Mac/iOS), TalkBack (Android), NVDA/JAWS (Windows). Interface "dibacakan" — images/icons/colors = invisible.

### 5.2 `sr-only` Utility — Visible untuk Screen Reader Only
Current: 9 instances, bisa diperbanyak untuk icon-only buttons.

**Pattern: Icon button with hidden label**
```tsx
<button aria-label="Hapus produk" className={...}>
  <Trash2 className="size-4" />
</button>
```

**Atau dengan visible-to-screen-reader text:**
```tsx
<button className={...}>
  <Trash2 className="size-4" aria-hidden="true" />
  <span className="sr-only">Hapus produk</span>
</button>
```

Kedua OK. `aria-label` lebih singkat, `sr-only` span bagus kalau text reused.

### 5.3 `aria-hidden` — Hide dari Screen Reader
Decorative elements (icons purely visual) wajib `aria-hidden="true"`:
```tsx
<button>
  <Sparkles className="size-4" aria-hidden="true" />
  <span>AI Forecast</span>
</button>
```

Icon diumumkan sebagai "sparkles, AI Forecast" tanpa hidden → redundant. Dengan hidden: "AI Forecast".

### 5.4 `aria-live` Region — Dynamic Content Announcement
**Currently 0 — critical gap for toast, loading updates.**

Per WCAG, setiap dynamic content update harus bisa diumumkan:
| Priority | When | Use |
|---|---|---|
| `aria-live="polite"` | Non-urgent (toast success, data refresh) | Most cases |
| `aria-live="assertive"` | Urgent (error, session expiring) | Sparingly |
| `aria-live="off"` | Disable announcement | Default |

**Sonner toast setup:** `sonner` library sudah pakai `role="status"` + `aria-live="polite"` — **ensure config** di `components/ui/sonner.tsx` tidak override.

**Custom loading pattern:**
```tsx
<div aria-live="polite" aria-busy={isLoading}>
  {isLoading ? "Memuat data..." : data}
</div>
```

### 5.5 Semantic Roles (Implicit > Explicit)
**Prefer semantic HTML** over role hack:
| ❌ Don't | ✅ Do |
|---|---|
| `<div role="button" onClick>` | `<button>` |
| `<div role="link" onClick>` | `<a href>` or `<Link>` |
| `<div role="heading">` | `<h1>..<h6>` |
| `<div role="list">` | `<ul>` or `<ol>` |
| `<div role="navigation">` | `<nav>` |
| `<span role="img">` | `<img alt>` or `<svg aria-label>` |

**Audit finding:** 6 `<div onClick>` instances = non-semantic clickables. **Refactor to `<button>`**.

### 5.6 Landmark Regions
Pakai semantic elements sebagai landmarks (screen reader can jump between):
```html
<header>    <!-- banner role -->
<nav>       <!-- navigation role -->
<main id="main-content">  <!-- main role -->
<aside>     <!-- complementary role -->
<footer>    <!-- contentinfo role -->
```

**Rule:** Hanya **1 `<main>` per page**. Audit finding: 3 `<main>` instances — OK jika di different routes.

## 6. Form Accessibility

### 6.1 `<label>` + `htmlFor` — WAJIB (Critical Gap)
**Audit: 51 `<label>`, only 1 `htmlFor` = 50 broken associations.**

Pattern:
```tsx
// ❌ Wrong — label not associated
<label className="...">Nama Produk</label>
<input type="text" className="..." />

// ✅ Correct — explicit association
<label htmlFor="product-name" className="...">
  Nama Produk
</label>
<input id="product-name" type="text" className="..." />

// ✅ Alternative — implicit association (nested)
<label className="...">
  <span>Nama Produk</span>
  <input type="text" className="..." />
</label>
```

**Why matters:** Click label → focuses input. Screen reader reads label saat input focused.

### 6.2 `aria-required` — Mark Required Fields
```tsx
<input
  type="email"
  required
  aria-required="true"
/>
```

`required` (HTML5) + `aria-required` (redundant tapi consistent across browsers/screen readers).

**Visual indicator pair:**
```tsx
<label htmlFor="email">
  Email <span className="text-rose-500" aria-hidden="true">*</span>
  <span className="sr-only">(wajib diisi)</span>
</label>
```

### 6.3 `aria-invalid` + Error Message Association
**Audit: 1 `aria-invalid` — major gap.**

Pattern:
```tsx
<label htmlFor="email">Email</label>
<input
  id="email"
  type="email"
  aria-invalid={!!error}
  aria-describedby={error ? "email-error" : undefined}
/>
{error && (
  <p id="email-error" className="text-xs text-rose-600" role="alert">
    {error}
  </p>
)}
```

**Why:**
- `aria-invalid="true"` — announces "invalid" saat focused
- `aria-describedby="email-error"` — reads error message
- `role="alert"` — immediate announcement when error appears

### 6.4 Fieldset + Legend untuk Group
Untuk radio group atau checkbox group:
```tsx
<fieldset>
  <legend className="text-sm font-bold">Metode Pembayaran</legend>
  <label>
    <input type="radio" name="payment" value="cash" />
    Tunai
  </label>
  <label>
    <input type="radio" name="payment" value="qris" />
    QRIS
  </label>
</fieldset>
```

Atau pakai `role="radiogroup"` + `aria-labelledby`:
```tsx
<div role="radiogroup" aria-labelledby="payment-label">
  <span id="payment-label" className="text-sm font-bold">Metode Pembayaran</span>
  {/* radio buttons */}
</div>
```

### 6.5 Helper Text via `aria-describedby`
```tsx
<label htmlFor="password">Password</label>
<input
  id="password"
  type="password"
  aria-describedby="password-help"
/>
<p id="password-help" className="text-xs text-slate-500">
  Minimal 8 karakter, gabungan huruf dan angka
</p>
```

### 6.6 Form Submission Error Summary
Setelah submit gagal dengan multiple errors:
```tsx
<div
  role="alert"
  aria-live="assertive"
  tabIndex={-1}
  ref={errorSummaryRef}   // auto-focus on submit
>
  <h3>Terdapat {errorCount} kesalahan:</h3>
  <ul>
    {errors.map(err => (
      <li key={err.field}>
        <a href={`#${err.field}`}>{err.message}</a>
      </li>
    ))}
  </ul>
</div>
```

## 7. Color Contrast

### 7.1 WCAG AA Minimum Ratios
| Content Type | Minimum Contrast |
|---|---|
| Body text (< 18pt) | **4.5:1** |
| Large text (≥ 18pt / 14pt bold) | **3:1** |
| UI components (borders, icons) | **3:1** |
| Decorative | No requirement |

### 7.2 InsightSphere Palette Compliance
(Refer ke `COLORS.md §7`)

✅ **Passes AA:**
- `text-slate-900` on `bg-white` (21:1)
- `text-slate-700` on `bg-white` (10:1)
- `text-slate-600` on `bg-white` (6.8:1)
- `text-slate-500` on `bg-white` (4.6:1) — minimum OK
- `text-indigo-600` on `bg-white` (6.9:1)
- `bg-indigo-600 text-white` (6.3:1)
- `text-emerald-600` on `bg-white` (5.3:1)

⚠️ **Borderline (check before use):**
- `text-slate-400` on `bg-white` (3.1:1) — **OK for large text only**
- `text-slate-300` on `bg-white` (1.9:1) — ❌ fails, use `text-slate-400` minimum

❌ **Fails AA:**
- `text-slate-200` / `text-slate-100` on `bg-white` — decorative only
- Light badge backgrounds with light text

### 7.3 Color Not Sole Indicator
```tsx
// ❌ Wrong — color-only status
<span className="text-red-600">Failed</span>

// ✅ Correct — color + icon + text
<span className="text-rose-600 flex items-center gap-1">
  <XCircle className="size-4" aria-hidden="true" />
  Gagal
</span>
```

Colorblind users (8% male, 0.5% female) bisa-nya bedain icon/text.

### 7.4 Focus Ring Contrast
Focus ring wajib kontras ≥ 3:1 terhadap bg. `ring-indigo-500` kontras 3.6:1 di light mode — OK. Di dark mode pakai `ring-offset-slate-900` agar tidak blend.

## 8. Motion & Animation

### 8.1 `prefers-reduced-motion` — Critical Gap
**Audit: 0 motion-reduce / motion-safe instances.** 

Beberapa user (vestibular disorders, migraines) perlu reduced motion. OS setting `prefers-reduced-motion: reduce` harus dihormati.

### 8.2 Tailwind Modifier
```tsx
<div className={cn(
  "transition-all",
  "motion-reduce:transition-none",   // disable motion if reduced
  "motion-safe:animate-bounce"        // only animate if safe
)}>
```

### 8.3 Global CSS Pattern (in `theme.css` atau `globals.css`)
```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

**Ini catch-all** — semua animasi auto-reduced. Tidak perlu manual per-component.

### 8.4 Animation Should Be
- **Subtle** — duration 150-300ms max (per `MOTION.md`)
- **Skippable** — no looping without stop
- **Non-blocking** — content usable mid-animation
- **Purposeful** — not decorative-only

### 8.5 Carousel / Auto-Advance
❌ **No auto-advancing carousel > 5s tanpa pause control.**
✅ Always provide pause button + indicators.

## 9. Images, Icons & Media

### 9.1 `<img>` Alt Text — Wajib
**Audit: 4 `<img>`, 2 alt = 50% gap.**

| Image Purpose | Alt Strategy |
|---|---|
| **Informative** (product photo) | Describe content: `alt="Foto produk Kertas HVS A4 80gr"` |
| **Decorative** (bg pattern) | Empty alt: `alt=""` (not omit!) |
| **Functional** (clickable logo) | Describe destination: `alt="Logo SmartStock"` |
| **Complex** (chart, diagram) | Short alt + long description reference |

### 9.2 SVG Icons
```tsx
// ❌ Decorative SVG without hiding
<Trash2 className="size-4" />   // screen reader reads "Trash 2"

// ✅ Decorative hidden
<Trash2 className="size-4" aria-hidden="true" />

// ✅ Meaningful SVG with label
<svg aria-label="Logo InsightSphere" role="img">
  ...
</svg>
```

**Lucide icons** tidak otomatis `aria-hidden` — developer responsibility.

### 9.3 Icon-Only Button
```tsx
// ❌ No label
<button><Trash2 /></button>

// ✅ aria-label
<button aria-label="Hapus produk">
  <Trash2 aria-hidden="true" />
</button>

// ✅ sr-only text
<button>
  <Trash2 aria-hidden="true" />
  <span className="sr-only">Hapus produk</span>
</button>
```

## 12. 🚫 Prohibited Patterns

| Pattern | Why | Replacement |
|---|---|---|
| `<div onClick>` without `role` + keyboard | Keyboard fail, screen reader fail | `<button>` semantic |
| `outline-none` tanpa `focus-visible:ring-*` | Invisible focus = keyboard fail | Always pair |
| `<label>` tanpa `htmlFor` | No input association | Always associate |
| `<img>` tanpa `alt` | Screen reader reads filename | Always `alt` (empty if decorative) |
| `alt="image"` / `alt="icon"` | Uninformative | Describe content or semantic |
| `tabindex` > 0 | Breaks natural order | Only 0 or -1 |
| Auto-advancing content tanpa pause | WCAG 2.2.2 fail | Pause control |
| Color-only status indicator | Colorblind fail | Color + icon + text |
| Focus removed tanpa replacement | Keyboard users lost | Visible focus always |
| `title="..."` sebagai tooltip | OS-dependent, untuk a11y weak | Use `<Tooltip>` from shadcn (OVERLAYS.md) |
| Animation tanpa `motion-reduce` respect | Vestibular fail | Global CSS media query |
| ARIA role redundant with HTML (`<button role="button">`) | Noise | Remove role |
| Wrong ARIA (`<input role="textbox">`) | Anti-pattern | Use semantic |
| Modal tanpa focus trap | Focus escapes | Use Radix `<Dialog>` |
| Dynamic content tanpa `aria-live` | Silent update | Wrap in live region |
| Skip link absent | Keyboard users stuck in nav | Add skip-to-content |
| Form error tanpa `aria-describedby` | Error not announced | Wire error ID |
| Required field tanpa `aria-required` | State hidden | Add aria-required |
| Placeholder as label | Disappears on focus, low contrast | Use real `<label>` |
| Heading level skip (h1 → h3) | Breaks document outline | Sequential h1 → h2 → h3 |

Per feature, minimum:

- [ ] **Tab order** — logical left-to-right, top-to-bottom?
- [ ] **Focus visible** — always see where I am?
- [ ] **Esc closes** — modals, dropdowns, popovers?
- [ ] **Enter/Space** — activates buttons?
- [ ] **Arrow keys** — work in menus, tabs, radios?
- [ ] **No keyboard trap** — can always escape?
- [ ] **Form labels** — click label focuses input?
- [ ] **Error messages** — announced when appear?
- [ ] **Color contrast** — chrome DevTools Lighthouse check
- [ ] **Zoom 200%** — still usable?
- [ ] **Screen reader** — NVDA or VoiceOver walkthrough

## 17. Code Review Rubric

4. ✅ **Label associated** — `htmlFor` on every label → **reject**
