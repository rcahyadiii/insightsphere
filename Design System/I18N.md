## 3. Technology Stack

### 3.1 Library: `next-intl`
Per Phase 5 architecture decision (see `FRONTEND_GAP_PLAN.md`).

**Why `next-intl`:**
- ✅ Native Next.js 14+ App Router support
- ✅ SSR + client hydration seamless
- ✅ ICU message format (pluralization, gender, number formatting)
- ✅ Type-safe with TypeScript codegen
- ✅ Active maintenance

**Alternatives considered:**
- ❌ `react-intl` — heavy, App Router issues
- ❌ `i18next` — older, less Next.js-native
- ❌ `next-translate` — maintenance concerns

### 3.2 File Structure (proposed)

```
frontend/
  messages/
    id.json           # Bahasa Indonesia (primary)
    en.json           # English
  src/
    i18n.ts           # next-intl config
    app/
      [locale]/
        layout.tsx    # locale-aware layout
```

### 3.3 Route Strategy

**Default:** `/dashboard` → locale from cookie/browser (default `id`)
**Explicit:** `/en/dashboard` → English override
**Fallback:** `id` (primary)

```ts
// i18n.ts
import { getRequestConfig } from "next-intl/server";

export default getRequestConfig(async ({ locale }) => ({
  messages: (await import(`../messages/${locale}.json`)).default,
}));
```

## 4. Translation Key Convention

### 4.1 Naming Pattern

**`[domain].[entity].[property]`** — dot-separated, lowercase, snake_case.

| Good | Bad |
|---|---|
| `auth.login.title` | `loginTitle` |
| `inventory.product.field.name` | `productName` |
| `common.action.save` | `saveButton` |
| `error.network.offline` | `offlineError` |

### 4.2 Domain Prefixes

| Prefix | Scope |
|---|---|
| `common.*` | Shared across features (buttons, statuses, dates) |
| `nav.*` | Navigation items, menu labels |
| `auth.*` | Login, signup, reset, 2FA |
| `dashboard.*` | Dashboard KPI, charts, widgets |
| `inventory.*` | Products, stock, categories |
| `pos.*` / `cashier.*` | POS checkout, cart, payment |
| `report.*` | Reports, analytics |
| `settings.*` | User profile, preferences, store |
| `user.*` | User management, RBAC |
| `shift.*` | Cash management, shifts |
| `xai.*` | XAI explanations |
| `mlops.*` | MLOps dashboard |
| `error.*` | Error messages |
| `success.*` | Success toasts |
| `warning.*` | Warning copy |
| `validation.*` | Form validation messages |

## 5. ICU Message Format

next-intl supports ICU format — pakai untuk variables, plurals, selects.

### 5.1 Variables

```json
{ "greeting": "Halo, {name}!" }
```
```tsx
t("greeting", { name: "Faiz" })  // "Halo, Faiz!"
```

### 5.2 Pluralization

Indonesian = simpler than English (mostly no plural form difference, but count context matters).

```json
{
  "itemCount": "{count, plural, =0 {Tidak ada item} one {# item} other {# item}}"
}
```

```tsx
t("itemCount", { count: 0 })  // "Tidak ada item"
t("itemCount", { count: 1 })  // "1 item"
t("itemCount", { count: 5 })  // "5 item" (ID: no plural suffix)
```

**English equivalent:**
```json
{
  "itemCount": "{count, plural, =0 {No items} one {# item} other {# items}}"
}
```

### 5.3 Select (gender, role)

```json
{
  "welcome": "{role, select, owner {Selamat datang, Pemilik!} admin {Selamat datang, Admin!} cashier {Selamat datang, Kasir!} other {Selamat datang!}}"
}
```

### 5.4 Number / Currency / Date Inline

ICU can format inline:

```json
{
  "revenue": "Pendapatan hari ini: {amount, number, ::currency/IDR}",
  "timestamp": "Transaksi pada {date, date, long}"
}
```

```tsx
t("revenue", { amount: 1500000 })  // "Pendapatan hari ini: Rp 1.500.000"
t("timestamp", { date: new Date() })  // "Transaksi pada 23 April 2026"
```

## 6. Locale-Specific Formatting

### 6.1 Currency (Rupiah)

**Use `Intl.NumberFormat`** consistently (replace 63 `toLocaleString` + 111 hardcoded `"Rp "`).

```ts
// frontend/src/app/lib/format.ts
export function formatRupiah(
  amount: number,
  opts?: { compact?: boolean; decimals?: number }
): string {
  if (opts?.compact) {
    return new Intl.NumberFormat("id-ID", {
      style: "currency",
      currency: "IDR",
      notation: "compact",
      compactDisplay: "short",
      maximumFractionDigits: 1,
    }).format(amount);
    // → "Rp 1,5 jt"
  }

  return new Intl.NumberFormat("id-ID", {
    style: "currency",
    currency: "IDR",
    minimumFractionDigits: opts?.decimals ?? 0,
    maximumFractionDigits: opts?.decimals ?? 0,
  }).format(amount);
  // → "Rp 1.500.000"
}
```

**Indonesian compact abbreviation** (via `Intl.NumberFormat`):

| Range | Output |
|---|---|
| 1.000 | `Rp 1 rb` |
| 1.500.000 | `Rp 1,5 jt` |
| 1.500.000.000 | `Rp 1,5 M` |

### 6.2 Dates

**Use `Intl.DateTimeFormat`** atau `date-fns` dengan locale `id`:

```ts
import { format, formatDistanceToNow } from "date-fns";
import { id, enUS } from "date-fns/locale";

const locale = currentLocale === "id" ? id : enUS;

format(date, "d MMMM yyyy", { locale })
// id: "23 April 2026"
// en: "23 April 2026" (same month name coincidentally)

format(date, "d MMM yyyy, HH:mm", { locale })
// id: "23 Apr 2026, 13:45"
// en: "23 Apr 2026, 1:45 PM" (may differ)

formatDistanceToNow(date, { locale, addSuffix: true })
// id: "5 menit yang lalu"
// en: "5 minutes ago"
```

### 6.3 Numbers

```ts
new Intl.NumberFormat("id-ID").format(1500)
// → "1.500"

new Intl.NumberFormat("en-US").format(1500)
// → "1,500"

new Intl.NumberFormat("id-ID", { style: "percent" }).format(0.85)
// → "85%"
```

### 6.4 Locale Reference Table

| Format | ID | EN |
|---|---|---|
| Currency | `Rp 1.500.000` | `IDR 1,500,000` (or `Rp 1,500,000`) |
| Date long | `23 April 2026` | `April 23, 2026` |
| Date short | `23 Apr 2026` | `Apr 23, 2026` |
| Time | `13:45` | `1:45 PM` |
| Number | `1.500,50` | `1,500.50` |
| Percent | `85%` | `85%` |
| Decimal separator | `,` (koma) | `.` (period) |
| Thousand separator | `.` (titik) | `,` (comma) |

## 7. Text Expansion Policy

**Indonesian ~30% longer than English** rata-rata. UI wajib handle expansion:

### 7.1 Examples

| EN (chars) | ID (chars) | Expansion % |
|---|---|---|
| "Save" (4) | "Simpan" (6) | +50% |
| "Cancel" (6) | "Batal" (5) | -17% |
| "Delete" (6) | "Hapus" (5) | -17% |
| "Settings" (8) | "Pengaturan" (10) | +25% |
| "Log in" (6) | "Masuk" (5) | -17% |
| "Forgot password?" (16) | "Lupa password?" (14) | -12% |
| "Last transaction" (16) | "Transaksi terakhir" (18) | +12% |
| "Add new product" (15) | "Tambah produk baru" (18) | +20% |
| "Low stock alert" (15) | "Peringatan stok rendah" (22) | +47% |

### 7.2 Design Rules

- **Buttons:** min-width `96px` (6 karakter), but **no max-width** — allow text to dictate
- **Table headers:** truncate with tooltip if > 20 chars
- **Card titles:** allow wrap (`line-clamp-2`)
- **Nav items:** shorten for mobile (`"Manajer Inv."` vs `"Manajer Inventaris"`)
- **Labels:** avoid fixed column widths in form layouts

### 7.3 Handling Long Text

```tsx
// Truncate with tooltip
<Tooltip>
  <TooltipTrigger asChild>
    <span className="truncate max-w-[200px]">
      {product.name}
    </span>
  </TooltipTrigger>
  <TooltipContent>{product.name}</TooltipContent>
</Tooltip>

// Line clamp (multi-line)
<p className="line-clamp-2">
  {description}
</p>

// Ellipsis for single line
<span className="truncate">{longText}</span>
```

## 8. Language Switcher UI Pattern

### 8.1 Location

| Location | Pros | Cons |
|---|---|---|
| **Header user menu** | Accessible anywhere | Hidden on mobile |
| **Footer** | Low clutter | User won't find |
| **Settings page** | Full context | Extra click |
| **Login page** | First impression | Limited scope |

**Recommendation:** **Header user menu** + **Login page** (dual placement).

### 8.2 UI Design

```tsx
<DropdownMenu>
  <DropdownMenuTrigger asChild>
    <button className={btn("ghost", "sm")} aria-label="Pilih bahasa">
      <Globe />
      <span>{locale === "id" ? "ID" : "EN"}</span>
    </button>
  </DropdownMenuTrigger>
  <DropdownMenuContent align="end">
    <DropdownMenuItem onClick={() => switchLocale("id")}>
      🇮🇩 Bahasa Indonesia
      {locale === "id" && <Check className="ml-auto" />}
    </DropdownMenuItem>
    <DropdownMenuItem onClick={() => switchLocale("en")}>
      🇬🇧 English
      {locale === "en" && <Check className="ml-auto" />}
    </DropdownMenuItem>
  </DropdownMenuContent>
</DropdownMenu>
```

### 8.3 Persistence

- **Cookie:** `NEXT_LOCALE=id` — survives sessions
- **User preference:** also saved di user profile (DB)
- **Priority:** user profile > cookie > browser `Accept-Language` > default `id`

## 9. RTL Support (Future)

**Status:** Not currently supported. Placeholder policy untuk future Arabic/Hebrew market:

### 9.1 Setup (when needed)

```tsx
// app/layout.tsx
<html lang={locale} dir={isRTL(locale) ? "rtl" : "ltr"}>
```

```ts
export const isRTL = (locale: string) =>
  ["ar", "he", "fa", "ur"].includes(locale);
```

### 9.2 Tailwind RTL-Friendly Patterns

Use **logical properties** (not left/right):

| ❌ Avoid | ✅ Use |
|---|---|
| `ml-4`, `mr-4` | `ms-4` (margin-start), `me-4` (margin-end) |
| `pl-4`, `pr-4` | `ps-4`, `pe-4` |
| `text-left`, `text-right` | `text-start`, `text-end` |
| `rounded-l-lg`, `rounded-r-lg` | `rounded-s-lg`, `rounded-e-lg` |
| `left-0`, `right-0` | `start-0`, `end-0` |

Tailwind 4 supports these natively.

### 9.3 Icons

Flip directional icons in RTL:
```tsx
<ArrowRight className="rtl:rotate-180" />
```

## 10. Translation Workflow

### 10.1 Adding New Copy

1. **Write in Bahasa Indonesia first** (primary locale).
2. **Define key** following §4 convention.
3. **Add to `messages/id.json`**.
4. **Add English translation** ke `messages/en.json` (fallback OK to start).
5. **Commit both files** together.

### 10.2 Translation Quality Tiers

| Tier | When |
|---|---|
| **Auto-translate (MT)** | Rapid prototyping, draft |
| **Human translator** | Production release |
| **Native reviewer** | Final polish |

**Current stage:** MT OK for English sampai professional translator engaged.

### 10.3 Review Checklist

Per translation verify:
- [ ] Length fits UI (within ±50% of reference)
- [ ] Terminology consistent (check against `TERMINOLOGY.md`)
- [ ] Tone matches (`CONTENT.md §3`)
- [ ] Placeholders preserved (`{name}`, `{count}`)
- [ ] Capitalization correct per locale
- [ ] Punctuation appropriate (EN: period; ID: sometimes omit)

## 12. Type Safety

### 12.1 Typed Translation Keys

next-intl supports TS codegen untuk type-safe keys:

```ts
// messages.d.ts (auto-generated)
declare interface Messages extends Record<string, unknown> {
  common: {
    action: {
      save: string;
      cancel: string;
      ...
    };
  };
  ...
}
```

Usage:
```ts
const t = useTranslations("common.action");
t("save")    // ✅ type-safe
t("sav")     // ❌ TS error
```

### 12.2 ICU Argument Types

```ts
t("greeting", { name: "Faiz" })     // string required
t("itemCount", { count: 5 })        // number required
```

Codegen should enforce argument shapes.

## 13. 🚫 Prohibited Patterns

| Pattern | Why | Fix |
|---|---|---|
| Hardcoded Indonesian strings di JSX | Tidak translatable | Use `t("key")` |
| Concatenating translated strings | Breaks grammar in other languages | Use ICU format dengan variables |
| `"Hello " + name + "!"` | Word order assumption | `t("greeting", { name })` |
| `count === 1 ? "item" : "items"` | English-centric plural | ICU plural format |
| Hardcoded `"Rp "` currency prefix | Non-localizable, inconsistent | `formatRupiah()` / ICU currency |
| `.toLocaleString()` tanpa locale arg | Uses browser default, inconsistent | `new Intl.NumberFormat("id-ID")` |
| `date.toString()` | Non-localized output | `format(date, ..., { locale })` |
| `left`/`right` CSS direction | Breaks RTL | `start`/`end` logical properties |
| Missing fallback translation | Shows translation key as text | Always populate primary locale |
| Inline emoji for flags | Platform-inconsistent rendering | Use SVG flag icons |
